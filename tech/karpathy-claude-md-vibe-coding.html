🦠 AI 코딩의 3대 고질병

카파시가 직접 짚은 반복 실수 패턴

① 잘못된 가정

묻지도 않고 혼자 해석해서 원치 않는 방향으로 200줄 뚝딱

② 코드 부풀리기

3줄로 끝날 일을 추상화·옵션·에러처리로 200줄 만들기

③ 멋대로 고치기

한 함수만 고쳐달랬더니 옆 코드·포매팅·주석까지 전부 손댐

처방전: 카파시의 65줄 CLAUDE.md는 이 세 가지 고질병을 막는 4가지 브레이크를 담고 있어요. GitHub 스타 10만 개, 출시 3개월 만에요.

📋 카파시의 4가지 원칙

고질병 ① 해결

원칙 1

코딩 전 먼저 생각하라

가정하지 말고, 헷갈리면 먼저 물어보기. 해석이 여러 개면 다 보여주고, 더 단순한 방법 있으면 반박도 하기.

효과: 10분 코딩 후 "처음부터 다시" 방지

고질병 ② 해결

원칙 2

단순하게 먼저

최소 코드로 해결. 시키지 않은 기능·추상화·설정 가능성 금지. 200줄이 50줄로 가능하면 반드시 다시 짜기.

기준: 시니어가 "너무 복잡하다" 할 것 같으면 단순화

고질병 ③ 해결

원칙 3

외과 수술처럼 정확하게

시킨 것만 고치고 인접 코드 절대 건드리지 않기. 모든 변경 줄은 사용자 요청으로 거슬러 올라갈 수 있어야 함.

효과: "어제 잘 됐는데 오늘 갑자기 안 됨" 방지

카파시 핵심

원칙 4

목표 중심으로 실행

모호한 명령 대신 검증 가능한 성공 기준 제시. "버그 수정해줘" 대신 "버그 재현 테스트 먼저, 그 다음 통과시키기".

핵심: 강한 기준 = AI 혼자 달릴 수 있음

🇰🇷 한국어 환경 추가 규칙 5가지

카파시 65줄로 못 잡는 것들 — 직접 개발하며 추가

1

콜론 대신 마침표

한국어 문장 끝을 콜론(:)이 아닌 마침표(.)로 끝내기

2

파일 헤더 주석

새 소스 파일 맨 위에 그 파일의 역할을 한 줄 주석으로

3

계획 + 체크리스트

큰 작업 전 계획서 → 체크리스트 → 컨텍스트 노트 3종 세트

4

테스트 후 완료 선언

"끝났습니다" 전에 무조건 테스트 한 번 돌리기

5

의미 단위 커밋

한 문장으로 설명 가능하면 커밋, 안 되면 분리하기
