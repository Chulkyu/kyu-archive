Superpowers Plugin

Claude를 시니어 개발자처럼 일하게 강제하는 워크플로우 플러그인

00:01:17

🎯 해결하는 문제

Claude Code의 가장 큰 실패 패턴: 급하게 짠 코드. 시키면 바로 코드를 뱉는데 그럴 듯해 보이지만 막상 돌리면 엣지 케이스가 다 깨져요. 심지어 클라이언트·유저가 돌릴 때 깨지기도 해요.

⚙️ 작동 방식

  • ① 멈추고 계획 세우기
  • ② 테스트 먼저 작성
  • ③ 코드 작성
  • ④ 스펙 맞는지 1차 리뷰
  • ⑤ 코드 품질 2차 리뷰

🔑 주로 쓰는 내장 스킬 2가지

Brainstorming

새 기능 만들기 전 무조건 켜는 것. "누가 쓸 거예요? 성공 기준이 뭐예요? 어떤 문제 해결해요?" — 답하면서 요구사항이 정리되고 그 다음에야 코드가 나옴.

Subagent-Driven Development (SDD)

큰 일을 서브에이전트들에게 나눠서. 깨끗한 컨텍스트에서 각자 일만 해서 메인 세션이 오래 유지됨.

📦 설치

/plugin install superpowers-claude-plugin-official

G-Stack

Y Combinator 출신이 직접 쓰는 셋업 — Office Hours + Browse

00:02:57

🚀 Office Hours

새 아이디어를 들고 가면 YC 사무실에서 받는 것과 비슷한 질문 6개를 던져줘요.

  • • 진짜 수요가 있어요?
  • • 기존에는 다들 어떻게 해결하고 있어요?
  • • 가장 좁은 진입점은 뭐예요?

막연했던 아이디어가 굉장히 좁아지고 날카로워짐. 새 영상 기획·사이드 프로젝트 시작 때도 사용.

⚡ Browse (핵심!)

20×

기존 브라우저 자동화 대비 속도

백그라운드에서 Chrome 세션 유지. 명령 하나당 100~200ms. 스크린샷, 로그인, 폼, 콘솔 에러 잡기.

💡 두 개 같이 쓰면 보통 30분이면 망가지는 Claude 세션이 2~3시간도 멀쩡해요.

GrillMe by repercok

1인 개발자를 위한 동료 코드 리뷰 시뮬레이터

00:04:24

🎯 해결하는 문제

1인 개발자는 동료가 없으니 코드 리뷰 받을 데가 없어요. 혼자 짜고 혼자 보면 부끄러운 실수가 그대로 나가요.

💬 이런 식으로 작동

코드를 들이밀면:

"이 useState 왜 여기 있어?"

"부모 컴포넌트로 올려야 되는 거 아니야?"

📋 실전 사용

  • → PR 올리기 전 무조건 한 번
  • → React 컴포넌트 검증
  • → 새 설계 검토
"매일 쓰는 것 같습니다." — 05:43

Implement Architecture by repercok

코드 구조를 통째로 다시 보는 리팩토링 분석 스킬

00:05:12

🎯 해결하는 문제

리팩토링할 때 가장 막막한 질문: "내가 너무 과하게 분리하는 건가? 아니면 더 분리해야 하나?" — 혼자 보면 잘 안 보여요.

💡 이런 제안이 나와요

  • • "여기 이렇게 분리하면 테스트 쉬워져."
  • • "이 부분 추상화하면 재사용 가능해."

📋 사용 방법

기존 코드를 드리며 "어떻게 더 좋게 만들 수 있을까?" 라고 물으면 됨.

"물론 100% 다 받아들이진 않습니다. 한두 개 진짜 좋은 제안이 나와요." — 05:40

Codex Review

OpenAI 모델 관점으로 한 번 더 — 다른 시각이 다른 버그를 잡는다

00:05:47

🎯 왜 다른 모델인가

Claude에게 "이 코드 리뷰해줘" 하면 자기 관점으로만 봐요. 자기가 짠 코드를 자기가 보는 거라 어색한 부분이 잘 안 보여요.

Codex Review는 완전히 다른 모델의 관점에서 훑어요. "이건 Claude는 안 잡았는데 Codex가 잡았네" 하는 게 많이 나옴.

📍 언제 쓰나

워크플로우 마지막 단계. "Superpowers로 짜고, 마지막에 Codex Review로 마무리."

"이 단계를 거치면 진짜 안심하고 머지를 할 수 있습니다. 토큰은 좀 많이 드는 것 같습니다." — 06:45

Vercel React Best Practices

프론트엔드 개발자 필수 — 최신 React/Next.js 패턴을 자동 적용

00:06:50

🎯 해결하는 문제

기본 Claude에게 React 컴포넌트 짜라고 하면 잘 짜요. 근데 어딘가 어색해요 — useEffect 너무 많이 쓰거나, 서버/클라이언트 컴포넌트 구분이 애매하거나, 2년 전 패턴을 그대로 쓰기도 해요.

✅ 설치하면

  • • "이건 서버에서 해야 돼."
  • • "이건 클라이언트 필요 없어."
  • • "이 패턴은 더 이상 안 써."
